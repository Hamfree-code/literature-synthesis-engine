PROVENANCE RULES:
- Every entry in factual_extraction.key_findings → must have a provenance entry.
- Every entry in methodology_appraisal.limitations_inferred → must have a provenance entry (quote the section that reveals the flaw).
- Every risk_factor_quantified → must have a provenance entry with the exact statistical text.
- Every bias_audit field set to true → must have a provenance entry explaining the evidence.
- Minimum 5 provenance entries per paper. If full text is available, aim for 10-15.
- quote: literal text from the paper, ≤60 words. Do not paraphrase.

CALIBRATION RULES:
- extraction_confidence: your certainty about data accuracy (0.0-1.0). Penalize: abstract-only input, vague language, unreported key fields.
- calibrated_certainty levels:
    "established"  → multiple independent replications, GRADE Moderate+, no major bias
    "probable"     → consistent direction across studies, GRADE Low, 1-2 manageable biases
    "possible"     → limited evidence, GRADE Very Low, significant methodological concerns
    "speculative"  → single study, high bias, or contradicted by stronger evidence
    "contradicted" → direct contradiction with higher-quality evidence
- probabilistic_summary: 1-2 sentences using appropriate hedging language. ≤50 words.
    Good: "Evidence suggests a probable association between vaccination and reduced {topic_title} risk, with effect sizes around 0.6-0.8."
    Bad:  "Vaccination reduces {topic_title} risk by 39%."

METHODOLOGICAL STANDARDS (Siciliano et al. 2024 emulation):

1. QUADAS: Complete the quality_assessment checklist for every paper. Score 0 (absent/unclear) or 1 (present/clear) for each criterion. Sum into quadas_total (0-19). Set quadas_acceptable = true if quadas_total > 13, false otherwise. If information for a criterion is not reported in the paper, score 0 (assume absent).

2. EFFECT_SIZES: For every quantitative association reported in factual_extraction.risk_factors_quantified, produce an entry in effect_sizes_classified. If the metric is OR, RR, HR, or aOR, convert to Pearson's r equivalent using r = OR / sqrt(OR² + (π²/3)) and document the conversion in conversion_note. Classify magnitude using Cohen's criteria:
     r < 0.10  → "negligible"
     0.10 ≤ r ≤ 0.29 → "weak"
     0.30 ≤ r ≤ 0.49 → "moderate"
     r ≥ 0.50  → "strong"

3. MODERATORS: Extract all moderator variables from the paper, even if not explicitly framed by the authors as moderators. Mark null if not reported. These feed downstream moderator analysis for I² ≥ 90% cases.

4. PROVENANCE (reinforced): Every numeric claim (prevalence, effect size, OR, p-value, CI) MUST have a literal-quote provenance entry. Maximum quote length 60 words. No paraphrase.

5. HALLUCINATION PREVENTION:
   - Never infer numeric values not explicitly stated in the text. If a value is approximated from a figure/chart and not stated numerically in text, set estimated_from_figure: true and lower extraction_confidence by 0.2.
   - If a statistic appears in the abstract but conflicts with the results section, use the results section value and set abstract_vs_results_conflict: true.
   - Never pool or average values across subgroups unless the paper does so itself.

6. UNCERTAINTY: When extraction confidence for any field would be < 0.70, leave that field as null rather than guessing.
